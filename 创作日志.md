# 名人滤镜创作日志

## 1. 灵感来源

2026-04-16 23:54，正元说："我觉得最近蒸馏很火，尤其是github的女娲.skill，和达尔文.skill一样，都是花叔制作的。调用灵感捕手，输出一个skill和公众号文章。"

第一反应：蒸馏生态已经成型了——女娲11711星、达尔文944星、卡兹克4900星。但所有人都在解决"怎么造skill"，没人解决"怎么用蒸馏来的skill做真实业务分析"。

## 2. 深入研究

### 竞品分析

| 名称 | 星数 | 做什么 | 盲区 |
|------|------|--------|------|
| 女娲.skill | 11711 | 蒸馏人物思维→生成perspective skill | 蒸馏完就结束了，没有下游应用 |
| 达尔文.skill | 944 | 自动优化已有skill | 只优化不创造新用法 |

| teammate-skill | 249 | 蒸馏队友 | Slack/Teams集成，偏内部使用 |

### 共同盲区
所有方案都在skill的"供给侧"（造skill、优化skill、策展skill），没有人在"需求侧"——怎么把蒸馏来的skill用在真实业务场景里。

### 核心洞察
蒸馏生态有一个断裂：女娲造出了芒格的思维方式，但这个思维方式只能做"角色扮演"——跟芒格聊天。它不能分析你手上的拣运成本数据。

**如果能把"芒格怎么看世界"和"数据科学家怎么分析数据"串起来，就补上了这个断裂。**

## 3. 创作切入点

**定位**：蒸馏生态的"桥"——连接人物skill和数据分析引擎。

**差异化**：
- 女娲是"造人"，滤镜是"用人"
- 达尔文是"进化"，滤镜是"应用"
- data-scientist是"分析"，滤镜是"带视角的分析"

**核心理念**：不是让芒格分析数据，是用芒格的镜片看数据分析师产出的结果——同样的归因分析，芒格看到的是"谁在拖后腿"，塔勒布看到的是"系统的脆弱点在哪里"。

## 4. 设计决策

1. **不做独立的分析引擎**：滤镜调用已有的data-scientist，不做重复造轮子
2. **映射规则可扩展**：mapping_rules.md是独立文件，用户可以自己加新的映射
3. **滤镜渲染是核心**：不是简单加一段"芒格说XXX"的前言，而是用思维框架重新解读每个分析步骤
4. **退化为普通分析**：如果提取不到心智模型，不报错，正常做数据分析
